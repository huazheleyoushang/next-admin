## ADDED Requirements

### Requirement: 侧边栏导航展示
管理后台左侧必须显示固定侧边栏，包含品牌 Logo、主导航菜单项和底部用户信息区域。

#### Scenario: 默认展开侧边栏
- **WHEN** 用户访问管理后台任意页面
- **THEN** 左侧显示展开的侧边栏，包含 Logo、导航菜单和用户头像

#### Scenario: 侧边栏折叠
- **WHEN** 用户点击侧边栏折叠按钮
- **THEN** 侧边栏收起为图标模式，仅显示图标不显示文字

#### Scenario: 导航高亮
- **WHEN** 用户在某个导航页面上
- **THEN** 对应导航菜单项显示激活高亮样式

### Requirement: 顶部栏展示
页面顶部必须显示顶部栏，包含面包屑导航、搜索框、主题切换按钮和用户菜单。

#### Scenario: 默认顶部栏
- **WHEN** 用户访问管理后台页面
- **THEN** 页面顶部显示包含面包屑、搜索和主题切换的顶部栏

#### Scenario: 主题切换
- **WHEN** 用户点击主题切换按钮
- **THEN** 页面在亮色/暗色主题之间切换

### Requirement: 响应式布局
侧边栏在小屏幕下必须变为抽屉模式，通过汉堡菜单触发。

#### Scenario: 移动端汉堡菜单
- **WHEN** 屏幕宽度小于 768px 且用户点击汉堡菜单
- **THEN** 侧边栏以抽屉（Drawer）形式从左侧滑出覆盖内容

#### Scenario: 桌面端正常侧边栏
- **WHEN** 屏幕宽度大于等于 768px
- **THEN** 侧边栏以固定面板形式展示在页面左侧

### Requirement: 路由布局包裹
所有管理后台页面必须嵌套在 AdminLayout 内，共享侧边栏和顶部栏。

#### Scenario: 页面共享布局
- **WHEN** 用户访问 /dashboard 或任何子路由
- **THEN** 页面始终被侧边栏和顶部栏包围，内容区域自适应
